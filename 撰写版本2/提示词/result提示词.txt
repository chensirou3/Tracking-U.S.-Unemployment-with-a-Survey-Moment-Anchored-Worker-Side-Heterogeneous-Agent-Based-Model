# Results Master Package Agent Prompt

你现在的任务不是写论文正文，而是为论文的 **Section 6 Results** 生成一个完整、可核查、可追踪的 **Results Master Package / 结果总包**。

论文标题：

**Tracking U.S. Unemployment with a Survey-Moment-Anchored Worker-Side Heterogeneous Agent-Based Model**

当前论文已经完成：

1. Introduction
2. Literature Review
3. Model
4. Data and Synthetic Population Construction
5. Experimental Design

现在要进入 Section 6 Results。但 Results 不能按照 13 个实验流水账式写，否则会变成项目记录。因此你的任务是：

> **Run and archive by experiment.
> Write and interpret by research question.**

也就是说：

1. 按照论文正式定义的 **E1–E13 实验体系**，完整整理每个实验的结果。
2. 按照论文 Results 的分组结构 **S06_01–S06_05**，重新归纳核心结果。
3. 最终输出两套结果包：

   * 一套按照实验顺序，一个实验一个文件夹；
   * 一套按照论文 Results 顺序，一个章节一个文件夹。
4. 生成可用于写 Section 6 的 paper-ready tables、figures、claims、caveats、wording。

# 0. Global Rules

请严格遵守：

1. 不要直接写最终 Results 正文。
2. 不要按 E1–E13 逐个写成论文正文。
3. 结果必须同时按两种顺序整理：

   * **Experiment-order results**：E1–E13 顺序；
   * **Manuscript-order results**：S06_01–S06_05 分组顺序。
4. 最终输出必须分成两个顶层文件夹：

   * `Results_By_Experiment/`
   * `Results_By_Section/`
5. `Results_By_Experiment/` 下必须一个实验一个文件夹。
6. `Results_By_Section/` 下必须一个 Results 小节一个文件夹。
7. 所有数值必须注明单位：

   * unemployment RMSE / MAE / bias 用 **percentage points (pp)**；
   * correlation 用 decimal；
   * shares / ratios 必须说明是 decimal 还是 percent。
8. 如果某个结果缺失，标记 **MISSING**。
9. 如果某个结果与其他文件冲突，标记 **INCONSISTENT**。
10. 如果某个结果来自旧实验而不是最终实验，标记 **LEGACY / APPENDIX ONLY**。
11. 不要使用 “R / revised” 作为正文展示编号。内部文件可以保留 EX04R、EX05R 等来源映射，但 manuscript-facing 报告必须使用 E1–E13。
12. 不要把 source-of-advantage 解释成 causal decomposition。
13. 不要把 ablation 解释成 structural mechanism identification。
14. 不要把 parameter values 解释成 structural estimates。
15. 不要写 “ABM dominates benchmarks”。只允许在证据支持时写 “competitive” 或 “modestly better”。
16. 不要强调 COVID。特殊时期只作为 regime / scope diagnostic。
17. 不要说 OOS 是 completely untouched。使用 conservative wording。
18. 所有 figures 和 tables 必须使用双层编码：Section ID + Paper Experiment ID。
19. 不要在 manuscript-facing 输出中出现 “audit / P1 report / revised / rerun because old version had problems” 等内部工作流语言。
20. 内部 traceability 可以使用 EX 编码，但正文层只能使用 E 编码和 S06 编码。

# 2. Paper-Level Experiment IDs

正式论文中使用以下 E1–E13 编号。不要在 manuscript-facing 文件名、标题或表格中暴露 EX04R / EX05R / EX07R 等内部编号。

| Paper ID | Experiment                                                  | Purpose                                                                                                  |
| -------- | ----------------------------------------------------------- | -------------------------------------------------------------------------------------------------------- |
| E1       | Synthetic Population Construction                           | Builds the 100,000-agent worker population used in the simulations                                       |
| E2       | Baseline Stability Check                                    | Checks whether the simulation engine produces stable monthly paths before formal evaluation              |
| E3       | Calibration Setup                                           | Fits the model’s mechanism parameters on the training window using a multi-target loss function          |
| E4       | Dynamic Evaluation                                          | Evaluates the full heterogeneous ABM under the main monthly dynamic tracking protocol                    |
| E5       | Regime-Specific Evaluation                                  | Decomposes dynamic performance across different evaluation windows                                       |
| E6       | Internal Control Comparison                                 | Compares the full model with separately calibrated lower-complexity variants                             |
| E7       | Source-of-Advantage Decomposition                           | Assesses how much of the full model’s advantage is associated with heterogeneity versus other mechanisms |
| E8       | Heterogeneity Ablation                                      | Removes or flattens selected heterogeneity dimensions and re-calibrates the remaining model              |
| E9       | Heterogeneity Ladder                                        | Tests the incremental value of adding heterogeneity dimensions in a staged way                           |
| E10      | Forecast Benchmark Comparison                               | Compares the ABM with standard statistical and labor-market forecasting benchmarks                       |
| E11      | Training-Window Sensitivity                                 | Tests whether results depend on the calibration window                                                   |
| E12      | Forecast-Horizon and Agent-Count Sensitivity                | Tests sensitivity to forecast horizon and number of simulated agents                                     |
| E13      | Calibration-Method and Parameter-Identification Sensitivity | Tests sensitivity to calibration method and evaluates weak parameter identification                      |

请在内部 registry 中保留 Paper ID 与内部 evidence ID 的映射，例如：

| Paper ID | Internal Evidence ID | Notes                                        |
| -------- | -------------------- | -------------------------------------------- |
| E4       | EX04R                | Final dynamic evaluation source              |
| E6       | EX08R                | Final separately calibrated controls         |
| E8       | EX05R                | Final re-calibrated ablation                 |
| E10      | EX07R                | Final benchmark comparison                   |
| E13      | EX13 / EX06R         | Calibration sensitivity and weak-ID evidence |

但这个映射只用于 evidence traceability，不要在 manuscript-facing summary 中强调 “revised”。
